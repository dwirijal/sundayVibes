## 2024-06-25 - Prevent Layout Thrashing in Interactive Components
**Learning:** Found a performance bottleneck in `src/components/Dock.tsx` where layout thrashing (Forced Synchronous Layout) was occurring. Inside the `calculateMagnification` function, which runs continuously on mouse and touch movement, the code was interleaving DOM reads (`getBoundingClientRect()`) and DOM writes (`style.setProperty()`) for each item in a loop. This forces the browser to recalculate layout multiple times per frame, significantly degrading animation performance.
**Action:** When animating or applying styles to multiple elements based on layout calculations, always batch DOM reads before performing any DOM writes. Phase 1: gather all dimensions and calculate required styles. Phase 2: apply all styles at once.
