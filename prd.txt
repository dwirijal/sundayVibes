1. Executive Summary
Sunday Vibes adalah platform digital multifungsi yang menyatukan seluruh layanan kreatif dan teknis dalam satu ekosistem terintegrasi. Platform ini melayani klien yang membutuhkan jasa event organizer, produk digital, persewaan alat dokumentasi, jasa desain, pengembangan web (termasuk WordPress), dan jual beli foto profesional.


Filosofi platform: one-stop creative platform — klien cukup datang ke satu pintu untuk semua kebutuhan kreatif mereka, mulai dari konsep event hingga foto produk yang siap publish.


1.1 Problem Statement
* Klien kesulitan mencari vendor terpercaya yang bisa menangani berbagai kebutuhan kreatif sekaligus
* Proses booking layanan kreatif masih manual, tidak transparan, dan memakan waktu
* Tidak ada platform lokal yang mengintegrasikan layanan event + digital + dokumentasi dalam satu UI
* Portfolio dan katalog layanan sulit diakses secara profesional


1.2 Solusi
* Website profesional dengan CMS yang mudah dikelola (Payload CMS)
* Sistem booking online untuk semua layanan
* Marketplace foto dengan lisensi yang jelas
* Dashboard klien untuk tracking project dan invoice
* Portfolio showcase yang menarik untuk memenangkan kepercayaan klien baru

11. Non-Functional Requirements
11.1 Performance
* First Contentful Paint (FCP): < 1.8 detik
* Largest Contentful Paint (LCP): < 2.5 detik
* Cumulative Layout Shift (CLS): < 0.1
* Time to First Byte (TTFB): < 600ms
* Foto marketplace: lazy-load + progressive image


11.2 Keamanan
* HTTPS enforced via Cloudflare
* Payload CMS admin di subdomain terpisah: cms.sundayvibes.id
* Rate limiting pada endpoint booking dan auth
* File download hanya via signed URL (expired dalam 24 jam)
* Validasi input dengan Zod di semua form
* CSRF protection via Payload built-in


11.3 SEO
* Setiap halaman memiliki unique title, meta description, OG image
* Structured data: LocalBusiness, Product, BlogPosting
* Sitemap.xml auto-generated dari Payload collections
* URL slug SEO-friendly, lowercase, hyphen-separated
* Image alt text wajib untuk semua foto


11.4 Aksesibilitas
* WCAG 2.1 Level AA compliance
* Keyboard navigable untuk semua interaktif elemen
* Color contrast ratio minimal 4.5:1 untuk body text
* ARIA labels pada semua ikon dan tombol tanpa label teks


11.5 Browser Support
* Chrome 110+ | Firefox 110+ | Safari 16+ | Edge 110+
* Mobile: iOS Safari 16+ | Chrome Android 110+

Appendix A — Vibe Coding Guidelines
Dokumen ini dibuat untuk mendukung workflow vibe coding dengan Claude Code. Berikut konvensi yang perlu diikuti:


A.1 File & Folder Convention
* Komponen di src/components/{feature}/ComponentName.tsx
* Page di src/app/{route}/page.tsx (App Router)
* Payload collections di src/collections/{Name}.ts
* Hooks custom di src/hooks/useName.ts
* Utils di src/lib/{util}.ts
* Types di src/types/{name}.ts


A.2 Naming Convention
* PascalCase: Komponen React, Types, Interfaces
* camelCase: Fungsi, variabel, props
* kebab-case: URL slug, CSS class custom, file name
* SCREAMING_SNAKE: Konstanta environment


A.3 Commit Convention
* feat: fitur baru
* fix: bug fix
* chore: dependency, config
* style: styling changes
* content: update konten CMS/copy


A.4 Claude Code Slash Commands yang Disarankan
* /build-page [nama halaman] — scaffold halaman baru dengan layout standard
* /add-collection [nama] — buat Payload collection baru dengan field umum
* /style-component [nama] — refactor komponen dengan Sunday Vibes design tokens
* /fix-a11y — scan dan fix accessibility issues di halaman aktif
